## ARCHITECTURE B — the living `.charter.md` (the big pivot this session)
Design of record: `docs/plans/02-architecture-b-living-document.md` (ADOPTED 2026-07-23; go/no-go spike

THE MODEL: the plan file (extension `.charter.md`, #16) is the SINGLE CONTAINER of review state. Human
input (answers to `:::question`, agent edits from annotations) is written back INTO the `.charter.md`.
Guardrails consumes it via a DUAL PATH:
  - INTERACTIVE `/plan-breakdown` (human session, Skill tool) reads the `.charter.md` DIRECTLY (`:::`
    blocks and all), guided by a single Charter-published `charter-format` skill, within a format-version
    range. Richer signal than flattened prose (spike proved it).
  - HEADLESS/AUTONOMOUS breakdown (harness-driven, no Skill tool) reads the RETAINED flattened
    `charter handoff` output (plain markdown) — so plain plans AND Guardrails' autonomous pipeline (#361)
    are UNAFFECTED. THE FLATTENER STAYS PERMANENTLY.

KEY DESIGN DECISIONS (David's, settled):
  - Extension `.charter.md` (not `.chart` — taken by rhythm-game charts; not naked `.charter` — loses
    markdown tooling). `.charter.md` keeps markdown affinity + signals the dialect.
  - Single-writer = the DRAFTING AGENT (server NEVER writes the file — would race the agent's editor).
    Human answers queue in-memory (AnswerStore); the agent applies them inline on drain via
    `QuestionResolution.Apply` (surgical JsonObject key-add, atomic temp+rename in the plan dir).
  - Resolved vs open `:::question`: an optional inline `answer` array in the JSON body (non-empty=resolved).
  - Catalog = REAL ONLY: `:::custom-html` promoted to a first-class BlockKind; `:::file-tree`/
    `:::annotated-code` STRUCK (vaporware, no renderer). Drift test binds charter-format <-> BlockKind/
    QuestionSpec + the format version.
  - Open `:::question` at RUN TIME is DIAL-GOVERNED (Guardrails #361): best-guess-with-forensic-trail
    below the dial, else escalate (exit EscalationsPending=4, async answer channel). NOT an unconditional
    halt, NOT exit 2, NO new gate type (their agent-needsHuman classifier already covers it).
  - Solo review SUPPORTED -> durability is critical-path: server persists its queue to a server-owned
    SIDECAR (not the .charter.md), + a `charter resolve <plan>` verb for the solo case; `--apply` becomes
    the only answer-drain path.
  - Compatibility = a frontmatter `charter-format-version: F` marker + `skillMin <= F <= skillMax` range
    check (charter-format declares [format-min, format-version]); ANY catalog change bumps the version
    (drift-test-enforced); cumulative/backward-compatible skill; absent marker => reject. NOT a Guardrails-
    binary pin. Charter may require a min charter-format version (David OK'd version coupling).
  - Charter's living-document `charter handoff` inline-answer bridge: EmitQuestion falls back to the inline
    `answer` (shipped in slice 1), so a resolved .charter.md flattens faithfully (not all-open).
