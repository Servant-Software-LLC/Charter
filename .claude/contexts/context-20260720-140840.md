KEY DECISIONS & RATIONALE (roads taken AND rejected):
- Charter did NOT start as markdown+directives. First sketch was "go totally MDX, keep my
  own OSS" (full MDX + Preact/React SSR). markdown+`:::` directives (Markdig) won AFTER a
  format-research pass: ~90% of MDX's benefit without dragging a JS compiler+framework into
  a codebase with neither. The real driver is LEANNESS.
- Load-bearing insight: commenting depends on the loopback SERVER, not the authoring FORMAT.
  "server vs static" and "HTML vs MDX vs markdown" are INDEPENDENT axes; annotations anchor
  to DOM. So: render markdown -> HTML at BUILD time -> serve through the same server. Client-
  side render was rejected (makes the HTML inert without a bundle — the property Lavish avoids).
- No standalone M0 spike: "a DAG is a specification, not evidence"; M0's risk-proofs folded
  into wave-1 acceptance guardrails.
- Distribution = C# native binary + Homebrew (Guardrails #353), over npx-TS and over a Go/Rust
  rewrite. Charter is greenfield and COULD have gone npx but deliberately matched the C# suite.
- Charter's CI/CD is a PORT of Guardrails' pipeline (already-debugged), why the first release
  dry-run went green immediately.
- Charter.Core is intentionally ONE implementation task (not split model/renderer/source-map)
  to avoid Guardrails' #176 transitive-compile diamond. A documented WEAK sizing, on purpose.

FIRSTMATE / GUARDRAILS / AUTONOMY:
- firstmate = agent-distribution system dispatching autonomous crewmates in isolated worktrees,
  escalating only real decisions; a tool it drives can't block on an interactive human loop. Two-tier
  skills: public skills/ vs private .agents/skills/ (metadata.internal:true). firstmate is Shell;
  gnhf is TypeScript.
- Autonomy dial three-layer model (Guardrails #361): (1) judgment call -> dial arbitrates halt-vs-
  best-guess by criticality/confidence; (2) transient hard blocker -> bounded wait + backoff; (3)
  permanent blocker / retries exhausted -> unconditional halt-and-escalate. "The dial governs
  judgment, not physics." The overwatcher (#269) applies the auto end — e.g. auto-resolving the
  mechanical mermaid resource gap (#373).
- Charter #7 is deliberately THIN: its one decision is render-and-persist vs serve, no mid-run gates
  -> no dial. Autonomous Charter = headless render + persist deliverable to disk + exit, WITHOUT the
  loopback comment server / long-poll.
- Durable intent beyond v1: a cross-orchestrator MDX-planning capability usable from gnhf + firstmate
  public tiers AND at Tricentis.
