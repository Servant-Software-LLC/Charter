KEY DECISIONS & RATIONALE (roads taken AND rejected):
- Charter did NOT start as markdown+directives. First sketch was full MDX + Preact/React SSR. markdown+`:::`
  directives (Markdig) won after a format-research pass: ~90% of MDX's benefit without a JS compiler+framework.
  The driver is LEANNESS.
- Load-bearing insight: commenting depends on the loopback SERVER, not the authoring FORMAT. Render markdown
  -> HTML at BUILD time -> serve through the same server. Client-side render rejected (inert HTML w/o a bundle).
- No standalone M0 spike ("a DAG is a specification, not evidence"); M0's risk-proofs folded into wave-1 gates.
- Distribution = C# native binary + Homebrew (Guardrails #353), over npx-TS and a Go/Rust rewrite. Greenfield
  but deliberately matched the C# suite. Charter's CI/CD is a PORT of Guardrails' pipeline.
- Charter.Core intentionally ONE implementation task (not split) to avoid Guardrails' #176 transitive-compile
  diamond. A documented WEAK sizing, on purpose.
